📋 使用步骤

  1. 手机 B 开启个人热点

    设置 → 个人热点 → 开启，记下 WiFi 名称和密码

  2. 手机 A 连接热点

    手机 A WiFi 设置 → 连接手机 B 的热点

  3. 手机 A 连接蓝牙音箱

    蓝牙设置 → 配对蓝牙音箱 → 连接成功

  4. 手机 A 打开采集端

    浏览器打开「采集端」网页 → 允许摄像头和麦克风权限 → 对准卡尺

  5. 手机 B 打开主控端

    浏览器打开「主控端」网页 → 添加设备 → 查看监控画面

  6. 开始使用

    查看多画面、按住 PTT 说话、观察红色卡尺读数

⚠️ 注意事项

浏览器要求

请使用 Chrome、Edge、Safari 等现代浏览器，不支持 IE。

HTTPS 限制

浏览器要求 HTTPS 才能调用摄像头和麦克风。如果是手机之间局域网使用，需要在浏览器地址栏输入 chrome://flags/#unsafely-treat-insecure-origin-as-secure 将页面地址添加为例外，或直接用 HTTPS 部署。

蓝牙音箱

采集端手机连接蓝牙音箱后，接收到的语音会通过音箱播放。确保蓝牙已配对且已连接。

❓ 常见问题

摄像头打不开怎么办？
确保浏览器已获得摄像头和麦克风权限。在手机「设置 → 应用 → 浏览器 → 权限」中检查。部分浏览器需要 HTTPS 环境才能使用摄像头。
两台手机怎么连接？
手机 B 开个人热点，手机 A 连上热点后，两台就在同一个局域网了。在主控端添加采集端设备名称即可。
卡尺读数准不准？
当前网页版使用模拟读数演示效果。如需真实识别，需要集成 OpenCV AI 模型，建议使用 Flutter 原生版获得最佳精度。
可以加更多采集端手机吗？
可以！主控端支持多画面（最多 9 路），每台采集端手机打开 collector.html 页面，在主控端逐一添加设备即可。
对讲延迟大吗？
局域网内延迟极低（通常 < 200ms），使用 PTT 模式可以有效避免回声和啸叫问题。
